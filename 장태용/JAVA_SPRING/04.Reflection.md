## **리플렉션에 대해 설명해 주세요.**

리플렉션이란 컴파일 타임에 어떤 타입의 클래스를 사용할지 모르고 런타임에 클래스의 타입이 결정되어 실행해야 하는 경우에 사용합니다.

## 꼬리질문

### 의미만 들어보면 리플렉션은 보안적인 문제가 있을 가능성이 있어보이는데, 실제로 그렇게 생각하시나요? 만약 그렇다면, 어떻게 방지할 수 있을까요?

네 그렇게 생각합니다. 리플렉션을 통해 private으로 선언한 필드나 메소드에 접근할 수 있으므로 이는 캡슐화를 위반하고, 데이터 노출이나 수정이 발생할 수 있습니다.

이를 방지하기위해 사용을 최소한으로 줄이고, Java Security Manager을 통해 reflection을 사용하지 못하도록 권한을 설정할 수 있습니다.

### 리플렉션을 언제 활용할 수 있을까요?

IntelliJ와 같은 IDE에서 제가 작성한 클래스의 메소드를 사용할 때 필요한 매개변수 등을 미리 알려주는 자동완성 기능에 사용될 수 있습니다. 해당 클래스 정보를 동적으로 가져와 메소드 명, 파라미터 타입, 반환 타입 등을 추출하여 알려줄 수 있습니다.
